How it works

A relay of agents, supervised by code.

Every check runs the same five stages. Model stages propose; code stages measure character offsets, validate schemas, and check cited evidence verbatim against what the model was actually shown.

  1. 1 · CODE PARSE

    Structured document ingestion.
  2. 2 · MODEL EXTRACT

    Claims and references identified.
  3. 3 · MODEL ×N VERIFY

    Each claim checked against relevant sources.
  4. 4 · CODE VALIDATE

    Code enforces evidence requirements.
  5. 5 · CODE REPORT

    Findings rendered with citations.

SOURCE TEXT NOT FOUND → FAILS VISIBLY — an unvalidated citation is downgraded automatically, and the failure shows on the report itself.

  1. 1 parsingcode

    Sandboxed text extraction; every character position tracked. So findings cite exact offsets, not vague areas.

  2. 2 extractingmodel

    Every checkable claim pulled as a verbatim quote. So nothing is paraphrased before it is judged.

  3. 3 verifyingmodel ×N

    Each claim judged against retrieved reference excerpts, in parallel. So a long contract still checks in minutes.

  4. 4 reviewingcodemodel

    Deterministic checks, evidence validation, severity grading. So fabricated evidence never reaches your report.

  5. 5 renderingcode

    One self-contained report file, fully auditable. So you can archive it, diff it, print it to PDF.

The trust architecture

Enforced in code. Not requested in prompts.

Four rules stand between the models and your report. None of them is a prompt. All of them are code, and every one fails in your favor.

§1 Evidence must be verbatim.

A conflict’s cited sentence is checked character-for-character against the excerpts the verifier was shown. No match — the finding is downgraded, automatically. Fabricated evidence cannot reach a report.

§2 Unverifiable is not verified.

Claims no reference document addresses are counted and displayed on every report. Silence about unknowns is not permitted.

§3 Injection findings outrank the model.

Instructions embedded in a document are detected by code and appended after the model review — a document cannot talk the pipeline out of reporting them.

§4 Malformed output fails loudly.

Every model response is validated against a strict schema. Failures are retried, then surfaced as warnings on the report itself. Nothing passes silently.

Self-hosted — in plain English

Your server. Your keys. Your documents stay home.

Confyro is software you run on a computer your company controls — not a website you upload contracts to. Getting started is three steps:

  1. 1

    Install it once

    One container on any server you control — a cloud machine, an office computer, anything that runs Docker. Your welcome email carries the exact copy-paste commands; whoever looks after your computers has it running in about ten minutes. After that, nobody touches a terminal again.

  2. 2

    Enter your key

    Open Confyro in your browser, go to the License page, and paste the activation key from your welcome email. On AI-included plans your AI access switches on by itself — nothing to set up, no AI account to create.

  3. 3

    Check your first document

    Upload your reference library — contracts, rate cards, policies — then drop in a document to check. Confyro cites the evidence for every finding, flags anything it cannot verify, and never edits your file.

Questions a careful buyer asks

Asked and answered. On the record.

Where do my documents go?

Nowhere. They are checked and stored on your own machine. You can have uploads deleted automatically after every check; the reports are kept.

Can the vendor read them?

No. On every plan, the software’s contact with us is the license check — your license key and an anonymous machine ID out, a signed license back. On AI-included plans it also reports numeric usage counters: pages, checks, token totals. Counters, never content — no document text, no filenames. Bring-your-own-key installs send only the license ping. We could not see your documents if we wanted to.

What does the AI see?

On AI-included plans your server sends document text directly to Anthropic using a dedicated key we provision for your company — the content never passes through our servers, and each customer runs in an isolated workspace. On Enterprise it reads your documents through an account your company owns, under your own provider agreement — or the AI runs on your own hardware, so document content stays entirely in-house.

Can it change or send anything?

No. It cannot edit, forward, or file your documents. It only marks problems — with the exact sentence that proves each one — and leaves every decision to a person.

What does it cost to run?

On Individual, Team, and Business, the subscription is the whole bill — AI usage is included, measured in checks, and you never create an AI account. On Enterprise, model usage runs on your own API keys at your provider’s rates; the product ships an evaluation harness that measures the real numbers on your own documents.

Can it run fully air-gapped?

Yes. Point it at a local vLLM or Ollama server for the models, activate it with a signed license file instead of the online check, and it runs with no outbound network connection at all.

What file formats does it check?

Word (.docx and legacy .doc), Excel (.xlsx), PowerPoint (.pptx and legacy .ppt), PDF — including scanned PDFs via OCR — and plain text or Markdown.

Do I need to be technical to install it?

You need a server and about fifteen minutes. If you (or anyone at your company) have installed anything with Docker before, the install will feel routine; if not, it is a single copy-paste job for whoever manages your machines, and your welcome kit walks through it step by step. After that one-time setup, everything is a web page in the browser — nobody else ever touches a terminal. A one-click installer for solo professionals is on the roadmap.

What happens if you disappear?

Your installation keeps running on your server through its licensed term — it does not depend on our infrastructure to check documents. Every report is a self-contained file, and everything the product knows lives in one directory you own and back up.
